# 02 Record

## 参照画像

`../../image/image_if/記録_if.png`

## 目的

種目を素早く探してタップできる、主導線の記録画面にする。

## 構成

- 日付セレクタ: `2026/5/4 今日`
- 部位チップ: 胸、背中、脚、肩、腕、腹筋、その他
- 検索バー: `種目名・筋肉部位で検索`
- 編集ボタン
- 3列種目カード: 線画イラスト、右上の小さなMAX、1行の種目名
- 追加カード: `種目を追加`
- 下部ナビ: 記録 active

## 表示テキスト

`MAX`, `種目を追加`

## 状態差分

検索結果なし、部位切替、スワイプヒント、ゲスト/ログインの種目一覧差分を維持する。線画がない種目は部位/器具ベースのフォールバックを出す。

## 関連画面への追従

記録カードの3列、正方形寄り、線画、MAX、1行種目名のルールは、削除モードと種目追加リストにも反映する。前回記録はこの画面には出さず、記録モーダルと履歴詳細に集約する。

## 実装対象

`RecordPage`, `BodyPartNavigation`, `BodyPartCard`, `DateSelector`, `FooterNav`

## 未対応許容範囲

線画の精密さはAI生成の限界があるため、マシン形状の修正は実機確認ベースで段階的に行う。
