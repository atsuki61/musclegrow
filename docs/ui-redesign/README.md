# MuscleGrow UI Redesign

## 方針

`docs/image/image_if` の画像を実装参照にする。記録速度、視認性、継続感を優先した「トレーニングログ端末」寄りのUIにする。ただし画像案の黒背景/青アクセントは見た目の方向性であり、実装では既存のライト/ダークモードとアクセントカラー変更に追従する。

## 共通トークン

- 背景: `--mg-bg`
- 面: `--mg-surface`, `--mg-surface-strong`
- 境界: `--mg-border`
- 補助テキスト: `--mg-muted`
- 影: `--mg-shadow`
- 角丸: `--mg-card-radius`
- アクセント: 既存の `--primary` を使用
- アクセント前景: 既存の `--primary-foreground` を使用
- 基本背景/文字: 既存の `--background`, `--foreground`, `--muted`, `--muted-foreground`, `--border` を尊重する
- 部位色: 既存の `--color-chest`, `--color-back`, `--color-legs`, `--color-shoulders`, `--color-arms`, `--color-core`, `--color-other` を使用

## テーマ対応の制約

- UI刷新で固定の青、黒、白を前提にしない。操作アクセントは `--primary`、文字は `--foreground` / `--muted-foreground`、面と境界は `--mg-*` または既存トークンを使う。
- ライト/ダークの両方で読めるコントラストを維持する。画像案がダークUIでも、ライトモードではカード面、境界、影をライト側トークンへ自然に切り替える。
- 部位別の色はハードコードせず、`--color-*` 系のCSS変数から出す。部位チップ、フォールバック、履歴カレンダー、グラフ凡例で同じ色体系を使う。
- 画像素材内の筋肉ハイライト色はラスタ画像なので完全なテーマ追従はしない。周辺UIのバッジ、枠、チップでテーマ/アクセント追従を担保する。

## 共通UI

- 下部ナビはガラス風の固定ナビ。中央の記録ボタンだけ大きくする。
- 部位チップは横スクロール。選択中は部位色の塗り、非選択は薄い面と枠。
- カードは暗い半透明面、細い境界、控えめな内側ハイライト。
- 記録カードはスマホ幅でも3列を基本にする。カードは正方形寄りにし、線画、MAX、種目名をカード内に収める。
- 種目名は省略せず、文字数に応じて縮小して1行表示する。長い名称でも右端の見切れを起こさない。
- 記録カード上には前回記録を出さない。前回記録、履歴、セット詳細は記録モーダルと履歴詳細で扱う。
- 記録カードのUIを変えた場合は、記録モーダル、履歴詳細、種目追加リスト、カスタム作成の確認表示も同じトーンへ追従させる。
- 画面内テキストはコード側で正確に持つ。画像生成内の文字は参考扱い。

## 種目イラスト

- スタイル: リアルな人物写真ではなく、暗色UIに乗るグレー線画と青い対象筋ハイライト。
- 保存先: `public/exercise-illustrations/{body-part}/`
- 初期対象: `docs/トレーニング.md` に記載された普段のメニュー全種目。
- 解決方法: `exercise.name` からUI用ヘルパーで画像パスを返す。
- 未生成種目: 部位/器具ベースのフォールバック表示。

## 実装優先度

2. 記録モーダルと種目追加
3. ホーム
4. 履歴
5. グラフ
