# MuscleGrow 💪

**筋トレの記録・可視化・継続をサポートする Web アプリケーション**

> 既存の筋トレアプリは「記録できるけど見づらい」という不満から開発。  
> Big3の目標達成率をひと目で確認でき、継続モチベーションを維持できる設計にこだわりました。

🔗 **デモ**: [https://musclegrow.vercel.app/](https://musclegrow.vercel.app/)

---

## 🚀 主な機能


| 機能            | 概要                                     |
| ------------- | -------------------------------------- |
| **Big3 進捗管理** | ベンチプレス・スクワット・デッドリフトの目標達成率をプログレスバーで表示   |
| **筋トレ記録**     | 種目・重量・回数・セット数を記録。前回記録の自動表示で入力の手間を削減    |
| **カレンダー履歴**   | 部位ごとに色分けされたカレンダーで過去のトレーニングを可視化         |
| **推移グラフ**     | 種目ごとの重量・回数の成長推移を時系列グラフで確認              |
| **体組成管理**     | 身長・体重・体脂肪率・筋肉量を記録。BMI に応じた動的スタイル表示     |
| **ゲストモード**    | ログイン不要で localStorage に記録。会員登録後にデータ自動移行 |
| **ダークモード**    | ダークモード＋カラーテーマのカスタマイズに対応                |

## 🏗 アーキテクチャ

```
Browser (Next.js App Router)
    │
    ├── Server Actions ──→ Drizzle ORM ──→ Supabase (PostgreSQL)
    │                                             │
    │                                        Better Auth
    │                                       (Google OAuth)
    └── localStorage (ゲストユーザーのみ)
              │
        ログイン時に Supabase へ自動移行 (GuestDataMigrator)

```

### 認証フロー

初期実装ではクライアントサイドの `useSession` を使用していましたが、Next.js のナビゲーションプリフェッチで不要なセッション取得リクエストが大量発生することに気づき、サーバーサイド認証へ全面移行しました。これにより不要なネットワークリクエストを削減し、パフォーマンスを改善しています。

## 🔧 開発で工夫・苦労したこと

### ① ゲストデータの自動移行

ログイン前後でデータが消えないよう、localStorage と DB のハイブリッド管理を実装しました。`GuestDataMigrator` を `protected` レイアウトに配置し、ログイン直後に一度だけゲストデータを DB へ移行する設計にしています。

### ② 認証方式のリファクタリング

開発中盤で、クライアントサイド認証（`useSession`）がページ遷移のたびに不要なリクエストを発生させていることに気づきました。サーバーサイドのキャッシュ関数に統一することで問題を解決しましたが、認証周りのリファクタリングは影響範囲が広く、慎重に進める必要がありました。

### ③ Drizzle ORM のマイグレーション管理

本番環境（Supabase）へのマイグレーション適用を GitHub Actions で自動化しています。開発初期はマイグレーションファイルの冪等性が担保されておらず、CI が複数回失敗しました。原因を調査してマイグレーションスクリプトを修正し、再現性のある自動デプロイを実現しました。

### ④ パフォーマンス最適化

- `Promise.all` による並列クエリ実行でデータ取得の待ち時間を短縮
- 重いコンポーネントを `dynamic import` で遅延読み込みに変更
- 認証セッション取得を共通キャッシュ関数に集約し、重複リクエストを排除

---

## 💡 今後の改善点

- **DRY 原則の徹底**: 一部コンポーネントに重複した定数・関数が存在するため、共通ユーティリティへの抽出が必要
- **テストカバレッジの拡充**: Vitest でビジネスロジックのテストを実装済みだが、コンポーネントテストや統合テストが不足
- **Supabase 無料枠の制約対応**: 7日間非アクティブで自動停止するため、定期的な起動維持の仕組みが必要

## 🚀 ローカル起動

```bash
# リポジトリのクローン
git clone https://github.com/atsuki61/musclegrow.git
cd musclegrow

# 依存関係インストール
pnpm install

# 環境変数設定（.env.example を参考に）
cp .env.example .env.local

# 開発サーバー起動
pnpm dev

```
